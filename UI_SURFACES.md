# Orignym - UI Surfaces

## 1. Core Applications (Next.js)

### Public Registry
- **Landing Page:** Search bar for discovering coined terms.
- **Search Results:** Paginated, filterable list of published claims.
- **Claim Detail Page:** Displays the term, intended meaning, verification evidence summary, and clear trust disclaimers.

### User Dashboard
- **Drafts:** List of in-progress coined-word claims.
- **Verification View:** Detailed breakdown of a verification run, showing exact, near, phonetic, and semantic conflicts.
- **Suggestions UI:** Interface for reviewing and accepting alternate coined-word suggestions.
- **Settings:** Profile and export management.

### Admin Panel
- **Moderation Queue:** Dashboard for reviewing reports and disputes.
- **Audit Logs:** View immutable system events.

## 2. UX Principles
- **Clarity:** Never mislead users about legal status. Use "evidence-backed" instead of "cleared".
- **Transparency:** The verification view must distinctly present evidence items and limitations.
