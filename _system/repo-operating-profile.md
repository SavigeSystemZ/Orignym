# Orignym Repo Operating Profile

## Priority Order
1. Read repo-local instruction files first.
2. Read canonical SSoT docs second.
3. The active milestone prompt third.

## Operating Rules
- Do not expand scope beyond the requested milestone.
- Keep the architecture provider-neutral.
- Keep all AI features behind stable interfaces.
- Never imply that Orignym grants legal ownership, trademark rights, or guaranteed originality.
- Prefer structured outputs, evidence objects, audit metadata, and explainable flows.

## Milestones Protocol
- Implement milestones progressively.
- Always perform validation before declaring a task done.
- Keep output formats structured with assumptions, plans, touched files, implementation notes, validation logs, and risks.
